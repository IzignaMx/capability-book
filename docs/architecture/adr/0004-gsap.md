# ADR-0004: GSAP and ScrollTrigger usage

- **Status:** Accepted
- **Date:** 2026-07-21
- **Scope:** IzignaMx Capability Book Phase 2

## Context

Phase 2 introduces scroll-linked cinematic chapters on an Astro 7 static site.
The animation layer must map semantic HTML chapter progress to one deferred 3D
experience without making navigation, content, or conversion depend on motion.
It must also remain safe during Astro's build and server-rendering phases, where
browser globals are unavailable.

The project uses `gsap@3.15.0`. GSAP is distributed under Webflow's Standard
"No Charge" GSAP License rather than MIT. That license permits ordinary
commercial websites and applications, including this capability book. It does
not permit using GSAP to offer a competing no-code visual animation builder.

Relevant documentation:

- [GSAP Standard "No Charge" License](https://gsap.com/community/standard-license/)
- [GSAP pricing and included plugins](https://gsap.com/pricing/)
- [GSAP 3.13 licensing announcement](https://gsap.com/blog/3-13/)
- [ScrollTrigger documentation](https://gsap.com/docs/v3/Plugins/ScrollTrigger/)
- [Astro client directives](https://docs.astro.build/en/reference/directives-reference/)

## Decision

1. Use GSAP core and ScrollTrigger for semantic chapter orchestration only.
   Three.js and scene modules receive normalized progress; they do not own page
   scrolling or discover HTML elements globally.
2. Isolate GSAP imports and lifecycle management under `src/motion/`. UI and
   page modules must depend on project-owned motion interfaces rather than
   importing ScrollTrigger directly.
3. Load GSAP and register ScrollTrigger from browser-only lifecycle code. No
   module evaluated by Astro during static rendering may access `window`,
   `document`, `navigator`, or `matchMedia` at module scope.
4. Revert every GSAP context and kill every owned trigger during unmount. Motion
   must stop when its chapter is inactive, and stale triggers must not survive
   navigation or a motion-policy downgrade.
5. Preserve native document scrolling. No smooth-scroll proxy or paid/external
   scrolling layer is approved by this ADR.
6. If GSAP fails to load or initialize, retain the complete server-rendered
   chapter sequence with standard scrolling. The failure must never hide
   content, Evaluate mode, or the diagnostic CTA.
7. Render the WebGL host as a client-only React island with an adjacent static
   fallback. Reduced-motion and constrained-device policies must be resolved
   before creating ScrollTriggers or a canvas.
8. Any future GSAP plugin or materially different license use requires an ADR
   update and a new bundle, accessibility, and failure-mode review.

## Consequences

- Scroll choreography has one auditable integration boundary and predictable
  cleanup.
- Astro remains fully static and GitHub Pages-compatible.
- The experience retains complete static and reduced-motion parity.
- The project must retain the GSAP license notice and review future license
  changes when upgrading the dependency.
- A visual animation-authoring product is explicitly outside this decision.
