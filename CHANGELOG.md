# Changelog

All notable changes to the IzignaMx Capability Book are documented here.

## Unreleased / Phase 2 vertical slice

### Added

- A deferred, single-canvas Explore narrative with signal acquisition, modular
  `IZ` assembly, six capability orbits, and semantic HTML parity.
- Evidence-backed spatial encounters for OmniSync and Hamburguesa Nómada with
  localized project classifications, proof points, case-study links, and
  contextual diagnostic actions.
- A bilingual quality scan, final conversion uplink, persistent motion control,
  and a deterministic static fallback for reduced motion or WebGL failure.
- Adaptive rendering quality based on device signals and measured 120-frame
  windows, with permanent downgrade-only DPR profiles.

### Quality and release gates

- Deterministic unit and browser coverage for scene lifecycle, one-canvas
  ownership, forward/reverse chapters, WebGL failure, reduced-motion parity,
  keyboard flow, diagnostic context, and adaptive quality.
- Build-graph accounting for critical JavaScript, the initial 3D bootstrap,
  deferred scene packages, fallback media, and approved exception paths.
- GitHub Actions gates for evidence, type and brand checks, production build,
  3D budgets, Playwright, Pa11y, and protected GitHub Pages deployment.

### Architecture decisions

- Approved GSAP use is isolated behind browser-only orchestration with owned
  cleanup and static failure behavior.
- Cinematic assets follow a primitives-first, Meshopt-default model pipeline;
  no hero GLB is transferred in this vertical slice.
